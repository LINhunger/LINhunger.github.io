Redis：初识介绍（一）

发表于 2017-11-20 | 分类于 Redis

Redis是一种就要键值对的NoSQL数据库，由string、hash、list、set、zset等多种数据结构和算法组成。Redis所有数据都存放在内存中，读写性能惊人。

一、Redis使用场景

（一）Redis可以做什么

1. 缓存

合理地使用缓存可以加快数据的访问速度，而且能够有效地降低后端数据源的压力。Redis提供键值过期时间设置，并且也提供了灵活控制最大内存和内存溢出后的淘汰策略。

2. 排行榜系统

Redis提供了列表和有序集合数据结构，合理地使用这些数据结构可以很方便地构建各种排行榜系统。

3. 计数器应用

例如视频网站的播放数、电商网站浏览数。为了保证数据的实时性，每一次播放和浏览都要做加1操作。Redis天然支持计数功能而且计数的性能非常好。

4. 社交网络

赞/踩、粉丝、共同好友/喜好、推送、下拉刷新等是社交网络的必备功能、由于社交网络的访问量通常比较大，而且传统的关系型数据库不太适合保存这些数据，Redis提供的数据结构可以相对比较容易的实现这些功能。

（二）Redis不可以做什么

站在数据规模的角度看，数据可以分为大规模数据和小规模数据，我们知道的Redis的数据时存放在内存中的，如果每天有几亿的用户行为数据，使用Redis存储的话，基本上是一个无底洞。

站在数据冷热的角度看，数据分为热数据和冷数据，热数据指的是会被频繁访问的数据，反之则为冷数据。例如对于视频网站来说，视频基本信息基本上各个业务线都是经常操作的数据，而用户的观看记录不一定是需要访问的数据。但从数据冷热的角度，视频信息是热数据，而用户观看记录为冷数据。冷数据放在Redis上是对内存的一种浪费，而对于一些热数据可以放在Redis中加速读写，也可以减轻后端的负载。

三、字符串

字符串类型是Redis最基础的数据结构。首先都是字符串类型，而且其他数据结构都是在字符串类型基础上构建的。

字符串的类型的值实际可以是字符串（JSON、XML）、数字（整数、浮点）、二进制（图片、音频、视频），但是值最大不能超过512MB。

（一）命令

  • ex seconds：为键设置秒级过期时间。
  • px milliseconds：为键设置毫秒级过期时间。
  • nx：键必须不存在，才能设置成功。
  • xx：键必须存在，才能设置成功。

4. 计数

incr命令用于对值做自增操作，返回结果分三种情况：

  • 值不是整数，返回错误。
  • 值是整数，返回自增后的结果。
  • 键不存在，按照值为0自增，返回结果是1。

很多存储系统系统和编程语言内部使用CAS机制实现计数功能，会有一定的CPU开销，但在Redis中完全不存在这个问题，因为Redis是单线程架构，任何命令到了Redis都要顺序执行。

五、列表

列表是来存储多个有序的字符的字符串，列表中的每个字符串称为元素，一个列表最多可以存储2^32-1个元素。在Redis中，可以对列表两端插入和弹出，获取指定范围的元素列表，获取指定索引下表的元素。它可以在我们开发中充当栈和队列的角色。

列表类型有两个特点：

  1. 列表的元素是有限的，这就意味着可以通过索引下标获取某个元素或者某个范围内的元素列表。
  2. 元素是可以重复地。

六、集合

集合类型也是用来保存多个字符串类型，但和列表类型不一样的是，集合中不允许有重复的元素，并且集合中的元素是无序的，不能通过索引下标获取元素。

Redis除了支持集合内的增删查改，同时还支持多个集合取交集、并集、差集等。

七、有序集合

有序集合保留了集合不能有重复成员的特性，但不同的是，有序集合中的元素可以排序。但是它和列表使用下标索引作为排序的依据不同的是，它给每个元素设置一个分数（score），作为排序的依据。

有序集合中元素（member）不能重复，但是score可以重复。

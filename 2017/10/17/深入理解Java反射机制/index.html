深入理解Java反射机制

发表于 2017-10-17 | 分类于 Java基础

Java反射机制可以让我们在运行期获得任何一个类的字节码。包括接口、变量、方法等信息。还可以让我们在运行期实例化对象，通过get/set方法获取变量的值。

反射的核心就是在JVM运行时才动态加载类或调用方法/访问属性，它不需要事先（写代码的时候或编译期）知道运行对象是谁。

Java反射框架主要提供以下功能：

  1. 在运行时判断这个对象所属的类；
  2. 在运行时构造任意一个类的对象；
  3. 在运行时判断任意一个类所具有的成员变量和方法（甚至可以调用private方法或成员）；
  4. 在运行时调用任意一个对象的方法。

注意：是运行期而不是编译期。

二、反射的基本运用

这里我们介绍一个基本反射功能的实现。

（一）获得Class对象

方法有三种：

1. 使用Class类的forName静态方法
2. 直接获取某一个对象的class
3. 调用对象的getClass()方法

（二）判断是否为某个类的实例

一般地，我们用instanceof关键字来判断是否为某个类的实例。同时我们也可以借助反射中的Class对象的isInstance()方法来判断是否为某个类的实例，它是一个Native方法：

instanceof用法：boolean = object instanceof class;

（三）创建实例

通过反射来生成对象主要有两种方式。

1. 使用Class对象的newInstance()方法

此处创建的对象是一个无参构造函数构造的，故原对象需要存在无参的构造方法。

2. 通过Class对象获取指定的Constructor对象

先通过Class对象获取指定的Constructor对象，在调用Constructor对象的newInstance()方法来创建实例。这种方法可以用指定的构造器构造类的实例。

（四）获取方法

获取某个Class对象的方法集合，主要有以下几个方法：

1. getDeclaredMethods()

getDeclaredMethods()返回类或接口声明的所有方法，包括私有方法，但不包括继承的方法。

2. getMethods()

getMethods()方法返回某个类所有公用（public）方法，包括继承类的公用方法。

3. getMethod

getMethod方法返回一个特定的方法，其中第一个参数为方法名称，后面的参数为方法的参数对应的Class对象。

获取类构造器的用法与获取方法的用法类似。主要是通过Class类的getConstructor方法得到Constructor类的一个实例，而Constructor方法类中有一个newInstance方法可以创建一个对象实例。

（六）获取类的成员变量信息

getFiled: 访问公有的成员变量。
getDeclaredField：所有已声明的成员变量。但不能得到其父类的成员变量。

三、反射的注意事项

  1. 反射会额外消耗一定的系统资源，因此如果不需要动态地创建一个对象，那么久不需要使用发射。
  2. 反射调用方法时会忽略权限检查，因此可能会破坏封装性。

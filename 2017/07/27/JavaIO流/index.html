JavaIO流

发表于 2017-07-27

Java IO 是一套用来读写数据的API(输入和输出)。大部分程序都要处理一些输入，并由输入产生输出。为此，Java提供了java,io包。

一、Java IO概述

一）输入和输出—数据源和目标媒介

术语“输入”和“输出”有时候会让人感到很疑惑，一个应用程序的输入，往往是另一个应用程序的输出。那么OutputStream流到底是出入到目的地的流程呢？还是一个输出的流呢？后面的学习中，我才了解到：输入和输出是针对内存而言的，输入流可以理解为向内存输入，输出流可以理解为从内存输出。

为了消除这个疑惑，我试着给输入和输出起一些不一样的别名，让它们从概念上与数据的来源和数据的流向相联系。

Java的IO包主要关注的是从原始数据源的读取以及输出原始数据到目标媒介。以下是最典型的数据源和目标媒介：

  1. 文件
  2. 管道
  3. 网络连接
  4. 内存缓存
  5. System.in, System.out, System.error(注：Java标准输入、输出、错误输出)

二）java IO的用途和特征

IO包下有很多不同的类，每个类负责不同的功能，各类用途汇总如下：

  1. 文件访问
  2. 网络访问
  3. 内存缓存访问
  4. 线程内部通信（管道）
  5. 缓冲
  6. 过滤
  7. 解析
  8. 读写文本（Reader & Writer）
  9. 读写基本数据类型（long int etc）
  10. 读写对象

一、操作系统中堆栈的区别

（一）操作系统中堆和栈的区别

栈：有操作系统自动分配释放，存放函数的参数值，局部变量的值等。其操作方式类似于数据结构中的栈；

堆：一般由程序员分配释放，若程序员不释放，程序结束时可能由OS回收，分配方式类似于链表。

（二）Java中堆栈的区别

栈和堆都是Java用来在内存中存储数据的地方。与C++不同，Java自动管理堆和栈，程序员不能直接的设置堆和栈。

  1. 在函数中定义的一些基本类型的变量和对象的引用变量都在函数的栈内存中分配。
  2. 当一段代码定义一个变量时，Java就在栈中就在栈中为整个变量分配内存空间，当超过变量的作用域后，Java就会自动释放掉该变量所分配的内存空间。

堆内存呢？

  1. 是用来存放new创建的对象和数组。
  2. 在堆中分配的内存，是由Java虚拟机的自动垃圾回收器来管理的。
  3. 在堆中产生了一个数组或对象后，还可以在栈中定义一个引用变量，这个变量的取值等于数组或对象在堆内存中的首地址，以后就可以在程序中使用栈的引用变量来访问堆中的数组或对象。

（三）java中变量在内存中的分配

  1. 类变量（static修饰的变量）：在程序加载时系统就为他在堆中开辟了内存，堆中的内存地址存放于栈以便于高速访问。静态变量的周期一直持续到整个系统关闭。
  2. 实例变量：当你使用Java关键字new创建一个对象时，系统在堆中开辟的并不是连续的内存空间而一般会是一堆零散的，然后根据零散的堆内存地址，通过哈希算法换算成一长串数字以表征这个变量在堆中的物理位置。实例变量在对它的引用丢失后，将被GC列入回收名单中。
  3. 局部变量：声明在某方法中，执行到它的时候在栈开辟内存，一旦局部变量脱离作用域，内存立即释放。

二、Spring IOC原理

IOC又叫做控制反转（Inversion of Control）.

（一）依赖倒置

什么是依赖倒置原则？

比如我们设计一辆汽车：先设计轮子，然后根据轮子大小设计底盘，接着根据底盘设计车身，最后根据车身设计整个汽车。这里就出现了一个依赖的关系。

这样的设计看起来没问题，但是可维护性却很低。假设完工以后，上司却突然说市场需求的变动，要把车子的轮子的设计都改大一码。这下就蛋疼了：因为我们是根据轮子的尺寸设计的底盘，轮子的尺寸一改，底盘的设计就该修改，那也就是需要一连串的修改——整个设计都得修改。

现在我们换一种思路。先设计汽车的大概样子，然后根据汽车的样子来设计车身，根据车身设计底盘，这样依赖关系就倒置过来了。

这时候，如果我们需要改动轮子的设计，就不需要动底盘、车身、汽车的设计了。

这就是依赖倒置原则——把原本的高层建筑依赖底层建筑“倒置”过来，编程底层建筑依赖高层建筑。高层建筑需要什么，底层需要实现这样的需求，但是高层并不管底层是怎么实现的。

（四）Spring IOC

对于Spring框架来说，就是由Spring来负责控制对象的生命周期和对象间的关系。在一个对象中，如果要使用另外的对象，就必须得到它（自己new一个，或者从JNDI中查询一个），使用完之后还要将对象销毁（比如Connection等）。那么IOC是如何做到的呢？

所有的类都会在Spring容器中登记，告诉Spring你是什么，你需要什么，然后Spring会在系统运行到适当的时候，把你要的东西主动给你，同时也把你交给其他需要你的东西。

所有类的创建、销毁都由Spring来控制，也就是说控制对象生命周期的不再是引用它的对象，而是Spring。对于某个对象而言，以前是它控制其他对象，现在是所有的对象都被Spring控制，这就叫控制反转。

六、Servlet 是如何处理请求的

通常当我们发送一个请求如http://localhost:8080/TomcatTest/login，容器是如何找到 LoginServlet 这个类来处理的呢？

其实 tomcat 中有一个 mapper 类，专门用来保存 tomcat 中的 container 容器中的子容器信息，当 org.apache.catalina.connector.Request 类进去 Container 容器之前，Mapper 将会根据这次请求的 hostname,contextpath 把这些信息设置到 Request 的 mappingData 属性中去，最后再去匹配相应的 Servlet 处理器。

到此为止 Servlet 已经可以帮我们完成所有工作了，但是现在的 web 应用很少直接将交互的全部页面交给 servlet 去实现，而是采用更加高效的 mvc 框架去实现，这些框架的基本原理就是把所有的请求都映射到一个 servlet 中，然后去实现 service 方法，比如后面要学习的 SpringMVC。
